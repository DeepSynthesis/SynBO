\begin{abstract}
\newline
\textbf{SynBO} (Synthetic Bayesian Optimization) is an intelligent reaction optimization tool designed specifically for synthetic chemists. It leverages Bayesian Optimization (BO) algorithms to help researchers find optimal reaction conditions with minimal experimental effort. This article provides a comprehensive introduction to SynBO, explaining its core principles and practical usage. We demonstrate how chemists can efficiently utilize SynBO through an integrated skill-based interface (via OpenClaw) to accelerate reaction optimization, reduce experimental costs, and discover optimal conditions typically within 50--80 experiments instead of hundreds. The tool supports both single-objective (e.g., yield maximization) and multi-objective optimization (e.g., yield and enantioselectivity), making it applicable to a wide range of synthetic chemistry challenges.

\vspace{1em}
\newline\textbf{Keywords:} Bayesian Optimization, Reaction Screening, Machine Learning, Synthetic Chemistry, High-Throughput Experimentation
\end{abstract}

\section{Introduction}
% ============================================================================

\subsection{The Challenge of Reaction Optimization}

Optimizing a new chemical reaction is one of the most time-consuming and resource-intensive tasks in synthetic chemistry. A typical reaction optimization process involves screening numerous combinations of reaction conditions:

\begin{itemize}
    \item \textbf{Catalysts}: Various organocatalysts or metal complexes
    \item \textbf{Solvents}: Polar, non-polar, protic, aprotic options
    \item \textbf{Bases/Additives}: Acids, bases, ligands, electrolytes
    \item \textbf{Temperature}: From cryogenic to elevated temperatures
    \item \textbf{Concentration}: Reactant and reagent concentrations
    \item \textbf{Reaction Time}: From minutes to days
\end{itemize}

The traditional approach, known as \textbf{OFAT} (One-Factor-At-A-Time), is fundamentally inefficient. Consider a typical scenario with 5 catalysts, 5 solvents, 4 bases, and 4 temperatures: this yields $5 \times 5 \times 4 \times 4 = 400$ combinations. Testing all combinations is clearly impractical in terms of time, materials, and labor.

\subsection{The SynBO Solution}

\textbf{SynBO} addresses this challenge by employing Bayesian Optimization (BO), a machine learning technique that intelligently guides experimental design. Like an experienced chemist, SynBO ``learns'' from previous experiments and ``predicts'' which conditions are most likely to succeed next. The key advantages include:

\begin{enumerate}
    \item \textbf{Efficiency}: Typically requires only 50--80 experiments to find optimal conditions
    \item \textbf{Intelligence}: Balances exploitation (trying promising conditions) and exploration (testing unknown areas)
    \item \textbf{Flexibility}: Supports single and multi-objective optimization
    \item \textbf{Accessibility}: User-friendly interface designed for chemists without machine learning expertise
\end{enumerate}

\section{System Requirements and Installation}
% ============================================================================

\label{sec:requirements}

\subsection{Python Environment}

SynBO requires the following Python environment:

\begin{itemize}
    \item \textbf{Python Version}: 3.12 or higher
    \item \textbf{Operating System}: Windows, macOS, or Linux
    \item \textbf{Memory}: 4 GB RAM minimum (8 GB recommended)
    \item \textbf{Disk Space}: 1 GB free space
\end{itemize}

To check your Python version, run:

\begin{lstlisting}[language=bash]
python --version
\end{lstlisting}

If you need to install Python 3.12+, please download it from the official Python website: \url{https://www.python.org/downloads/}.

\subsection{Installation Methods}

There are two recommended ways to install SynBO:

\subsubsubsection{Method 1: Using pip (Recommended)}

\begin{lstlisting}[language=bash]
pip install synbo
\end{lstlisting}

\subsubsubsection{Method 2: Using OpenClaw with Skill}

The easiest way to use SynBO is through the \textbf{OpenClaw} platform with the built-in SynBO Skill. This method requires no manual installation and provides an interactive interface. See Section~\ref{sec:skill_interface} for detailed instructions.

\section{Using SynBO Through OpenClaw}
% ============================================================================

\label{sec:skill_interface}

\subsection{What is OpenClaw?}

\textbf{OpenClaw} is an AI-powered platform that provides access to various scientific skills, including SynBO for reaction optimization. Through OpenClaw, you can interact with SynBO using natural language without writing any code.

\subsection{Accessing the SynBO Skill}

To use SynBO through OpenClaw:

\begin{enumerate}
    \item Visit the OpenClaw platform
    \item Locate and select the \textbf{SynBO} skill from the skill library
    \item The skill will guide you through the optimization process interactively
\end{enumerate}

The SynBO Skill in OpenClaw provides a structured workflow that handles all the technical details for you. You simply provide your reaction data and optimization goals, and the system generates recommended experimental conditions.

\subsection{Skill Capabilities}

The SynBO Skill provides the following capabilities:

\begin{itemize}
    \item \textbf{Project Setup}: Initialize your optimization project with custom working directory and project name
    \item \textbf{Reaction Space Definition}: Input or upload your catalyst, solvent, base, and other reagent options
    \item \textbf{Descriptor Generation}: Automatically generate molecular descriptors for your reagents using RDKit
    \item \textbf{Initialization}: Generate the first batch of experimental conditions to test
    \item \textbf{Optimization}: Recommend new experimental conditions based on previous results
    \item \textbf{Analysis}: Identify problematic reagent combinations using AI-powered analysis
\end{itemize}

\section{Practical Workflow for Chemists}
% ============================================================================

\label{sec:workflow}

\subsection{Overview}

SynBO provides a skill-based interface that guides chemists through the optimization process in a structured manner. The workflow consists of four main phases:

\begin{enumerate}
    \item Project Setup and Configuration
    \item Reaction Space Definition
    \item Descriptor Generation (Optional)
    \item Iterative Optimization
\end{enumerate}

The system will guide you through each step. Below is a detailed description of what to expect at each phase.

\subsection{Phase 1: Project Setup}

When you first use the SynBO Skill, you need to set up your project:

\subsubsubsection{Working Directory}

Choose a folder on your computer where all project files will be stored. This includes:
\begin{itemize}
    \item Reaction space definitions
    \item Molecular descriptors
    \item Optimization results
    \item Experimental data
\end{itemize}

\subsubsubsection{Project Name}

Give your project a descriptive name (e.g., ``asymmetric\_hydrogenation\_optimization''). This name will be used to organize your files.

\subsubsubsection{Optimization Goals}

Specify what you want to optimize. For example:

\begin{itemize}
    \item \textbf{Single objective}: Maximize yield
    \item \textbf{Multi-objective}: Maximize yield AND enantioselectivity (ee)
    \item \textbf{With priorities}: Maximize yield (weight 1.0) and ee (weight 2.0)
\end{itemize}

Define the expected numerical range for each metric (e.g., yield: 0--100\%).

Example optimization settings:
\begin{lstlisting}[language=json]
{
    "optimization_settings": {
        "opt_metrics": ["yield", "ee"],
        "opt_direct_info": [
            {"opt_direct": "max", "opt_range": [0, 100], "metric_weight": 1.0},
            {"opt_direct": "max", "opt_range": [0, 100], "metric_weight": 2.0}
        ]
    }
}
\end{lstlisting}

This configuration tells the system to optimize both yield and enantioselectivity, with higher priority given to ee.

\subsection{Phase 2: Reaction Space Definition}

The reaction space defines all possible condition combinations you want to screen. You can provide this data in two ways:

\subsubsubsection{Method 1: Direct Input}

Provide SMILES strings for each reagent category directly through the chat interface. For example:

\begin{itemize}
    \item Catalysts: Pd(OAc)$_2$, Pd(PPh$_3$)$_4$, Pd$_2$(dba)$_3$, etc.
    \item Solvents: THF, Dioxane, Toluene, DMF, MeCN
    \item Bases: Cs$_2$CO$_3$, K$_2$CO$_3$, NaOEt, DBU, Et$_3$N
    \item Temperatures: 25, 50, 80, 100 °C
\end{itemize}

\subsubsubsection{Method 2: File Upload}

Upload CSV files with the following structure:

\begin{lstlisting}[language=csv]
SMILES,name
CCO,ethanol
CCCO,propanol
CCCOH,isopropanol
\end{lstlisting}

\textbf{Naming Convention}: If you don't provide names, the system automatically assigns names using the format ``\textit{reagent\_type}-1'', ``\textit{reagent\_type}-2'', etc.

Each reagent type should be saved as a separate CSV file (e.g., \texttt{catalyst.csv}, \texttt{solvent.csv}) in the project's reaction space folder.

\subsubsubsection{Special Values}

For categorical options like ``no catalyst'' or ``no additive'', you can use text placeholders such as:
\begin{itemize}
    \item \texttt{no\_catalyst}
    \item \texttt{none}
    \item \texttt{blank}
\end{itemize}

These will be properly handled by the system as zero vectors in the descriptor space.

\subsubsubsection{Continuous Variables}

For non-molecular variables (temperature, time, concentration), format as:

\begin{lstlisting}[language=csv]
Name,value
temperature,25
temperature,50
temperature,80
time,12
time,24
\end{lstlisting}

\subsection{Phase 3: Descriptor Generation}

Molecular descriptors are numerical representations of your reagents that enable the optimization algorithm to learn patterns. If you provide SMILES strings, the system can automatically generate descriptors using RDKit.

\subsubsubsection{Automated Generation}

When you request descriptor generation, the system will:

\begin{enumerate}
    \item Verify that reaction space data exists
    \item Confirm which reagent categories need descriptor calculation
    \item Compute RDKit descriptors (200+ molecular properties)
    \item Handle any invalid SMILES by using placeholder values (0.0)
\end{enumerate}

\subsubsubsection{When Descriptors Are Ready}

Once descriptors are generated, they will be saved in the project's molecular descriptor folder, ready for the optimization phase.

If you prefer, you can also provide pre-calculated descriptors in CSV format.

\subsection{Phase 4: Initialization}

When starting a new optimization project with no previous experimental data, the system generates an initial batch of conditions to test.

\subsubsubsection{What Happens}

The system uses smart sampling strategies (such as Latin Hypercube or K-means clustering) to select a diverse set of initial conditions that cover the reaction space evenly.

\subsubsubsection{Batch Size}

For initialization, we recommend testing 5--10 experiments. This provides enough data for the algorithm to start learning patterns.

\subsubsubsection{Output}

After initialization, you will receive an Excel file containing the first batch of recommended experimental conditions, including:
\begin{itemize}
    \item Catalyst selection
    \item Solvent choice
    \item Base/additive selection
    \item Temperature
    \item Other relevant parameters
\end{itemize}

\subsubsubsection{Next Steps}

\begin{enumerate}
    \item Download the Excel file with recommended conditions
    \item Perform these experiments in your laboratory
    \item Collect your results (yields, selectivities, etc.)
    \item Return to the SynBO Skill with your results
\end{enumerate}

\subsection{Phase 5: Iterative Optimization}

After completing initial experiments, you continue the optimization loop:

\subsubsubsection{Upload Results}

Provide your experimental results in CSV format. The file should contain:

\begin{itemize}
    \item Reagent columns (matching your reaction space)
    \item Results columns (e.g., \texttt{yield}, \texttt{ee})
    \item Optional: batch number to track experiment rounds
\end{itemize}

Example:
\begin{lstlisting}[language=csv]
catalyst,solvent,base,temperature,yield,ee
Pd(OAc)2,THF,Cs2CO3,25,45.2,82.1
Pd(PPh3)4,Dioxane,K2CO3,50,67.8,91.3
\end{lstlisting}

\subsubsubsection{Get Recommendations}

The system analyzes your results and recommends the next batch of experiments. The algorithm intelligently balances:

\begin{itemize}
    \item \textbf{Exploitation}: Testing conditions similar to your best results
    \item \textbf{Exploration}: Testing conditions in less-explored areas of the reaction space
\end{itemize}

\subsubsubsection{Batch Size}

For subsequent rounds, we recommend 1--5 experiments per iteration. This allows the algorithm to continuously learn while keeping experimental workload manageable.

\subsubsubsection{Repeat}

Iterate this process until you achieve satisfactory results:
\begin{enumerate}
    \item Perform recommended experiments
    \item Collect results
    \item Upload results to the system
    \item Get new recommendations
\end{enumerate}

Typically, 50--80 total experiments are sufficient to find optimal conditions.

\section{Case Studies}
% ============================================================================

\label{sec:case_studies}

\subsection{Case 1: Asymmetric Hydrogenation}

\textbf{Background}: Screening chiral phosphoric acid catalysts for imine asymmetric hydrogenation.

\textbf{Reaction Space}:
\begin{itemize}
    \item 6 catalysts (CPA-1 through CPA-6)
    \item 4 additives (MsOH, TfOH, TFA, None)
    \item 4 solvents (DCE, PhCF$_3$, Toluene, Et$_2$O)
    \item 4 temperatures (-20, 0, 25, 40 °C)
    \item 4 H$_2$ pressures (1, 10, 20, 50 atm)
\end{itemize}

\textbf{Total Combinations}: $6 \times 4 \times 4 \times 4 \times 4 = 1536$

\textbf{Optimization Objective}: High yield + High ee

\textbf{Result}: Only 24 experiments needed (vs. 1536 full combinations), found conditions with 94\% yield and 98\% ee.

\subsection{Case 2: Buchwald-Hartwig Amination}

\textbf{Background}: Pd-catalyzed aromatic amination, screening ligand and base combinations.

\textbf{Workflow}:
\begin{enumerate}
    \item Defined reaction space with multiple ligands and bases
    \item Ran initialization with 8 experiments
    \item Iteratively optimized for 5 more rounds
    \item AI analysis identified problematic ligand-base combinations
\end{enumerate}

\textbf{Outcome}: Automatically excluded problematic conditions, saving significant experimental time and resources.

\section{Best Practices and Tips}
% ============================================================================

\label{sec:best_practices}

\subsection{Experimental Design}

\begin{enumerate}
    \item \textbf{Initial Batch Size}: Start with 5--10 experiments for initialization
    \item \textbf{Subsequent Batches}: Use 1--5 experiments per optimization round
    \item \textbf{Replicates}: Include 2--3 replicate experiments to estimate experimental variability
    \item \textbf{Diversity}: Ensure initial conditions cover the full reaction space
\end{enumerate}

\subsection{When to Stop}

Monitor the following indicators:

\begin{itemize}
    \item \textbf{Hypervolume Plateau}: The algorithm's improvement slows significantly
    \item \textbf{Yield Saturation}: Recommended conditions consistently give similar yields
    \item \textbf{Resource Constraints}: When time or material budget is exhausted
    \item \textbf{Target Achievement}: When desired yield/selectivity is reached
\end{itemize}

Common stopping criteria:
\begin{itemize}
    \item Achieved target yield (e.g., $>$90\%)
    \item Achieved target enantioselectivity (e.g., $>$95\% ee)
    \item No significant improvement over 3--4 consecutive rounds
    \item Experimental resources exhausted
\end{itemize}
